# AI Experience Pattern Reference Images

Quick reference guide for the extracted design pattern images. All images are from the official Red Hat "Design language: AI-enabled features" deck (February 2026).

## Transparency Requirements

### `transparency-requirement.png` (Page 5)
**"It should always be clear when and how AI is being used"**

Key points:
- Don't rely on just one form of indicator
- At minimum use one visual and one verbal indicator
- Options: icons, "with AI" button text, animations, text disclaimers
- Consider additional indicators for high-risk interactions

### `transparency-components.png` (Page 6)
**Transparency components for AI-related features**

Shows notification message formats:
- Warning style: "[Feature name] uses AI. Check for mistakes"
- Info style: "Important - This feature uses AI technology"
- Icon placement before disclosure
- Privacy statement links
- Feedback mechanisms

### `transparency-notices-features.png` (Page 7)
**Transparency notices for AI-assisted features**

Examples of notice placement at beginning of AI-assisted experiences:
- Inline notices with AI-generated content
- Warning/info banners above features
- Full text of required notices

### `transparency-notices-chatbot.png` (Page 8)
**Transparency notices for virtual assistants**

Chatbot-specific transparency:
- Notice shown before user interacts
- "This feature uses AI technology. Do not include any personal information..."
- Persistent notice: "Always review AI-generated content prior to use"
- Visual indicator (AI icon) + verbal indicator ("AI" tag)
- Side-by-side light/dark examples

### `ai-generated-content-label.png` (Page 9)
**Indicating AI-generated content**

Examples of labeling AI-generated search results and content:
- Sparkle icon + "AI-assisted results" heading
- Visual + verbal indicators together
- Content presentation patterns

## Color & Styling

### `color-no-gradients.png` (Page 22)
**Don't use color or gradients to indicate AI**

Two-column explanation:

**AI features use same colors as other elements:**
- Avoids overwhelming UI when AI is ubiquitous
- Color palette designed for status and accessibility

**Do not use gradients and color coding:**
- Avoids conflicting with status colors
- Avoids accessibility challenges
- Avoids confusion with marketing materials

### `color-dos-donts.png` (Page 24)
**Color do's and don'ts examples**

Three examples side-by-side:
- ✅ **Do this**: Standard colors - "Edit with AI" button in standard blue
- ❌ **Not this**: Purple/gradient colors
- ❌ **Not this**: Gradient fill on button

Clear visual comparison of correct vs. incorrect color usage.

## Chatbot Avatars

### `chatbot-avatar-robot.png` (Page 26)
**Use the new robot icon for chatbot avatars**

Standard icon and UI icon versions shown.

**Why robot for chatbots:**
- Creates connection with chat expectation (validated by survey)
- Cohesive across Red Hat chatbots

**Why this robot specifically:**
- More recognizable and expressive (validated by Qualtrics survey)
- Conveys advanced AI capabilities

### `chatbot-avatar-variations.png` (Page 27)
**Chatbot avatar variations**

Four emotional states:
- Happy chatbot (default)
- Annoyed chatbot
- Bewildered chatbot
- Sad chatbot

Use for personality, reactions, or differentiating options.
Note: UI icon has no variations due to size constraints.

### `chatbot-avatar-in-use.png` (Page 28)
**Chatbot avatars in use**

Color variants and sizing:
- Light theme: 8 color options (teal, red, pink, orange, yellow, purple, gray, white)
- Dark theme: 8 corresponding color options
- Three size variants: small, medium, large
- Using non-status color tokens

Shows robot icon used in actual chat interface.

### `chatbot-launch-button.png` (Page 29)
**Chatbot launch button states**

Three button variants with states:
1. **Primary icon button**: Default, Hover, Clicked/Expanded (light & dark)
2. **Secondary icon button**: Default, Hover, Clicked/Expanded (light & dark)
3. **Inverse icon button**: Default, Hover, Clicked/Expanded (light & dark)

Each shows robot icon in default state, then chevron when expanded.

### `chatbot-launch-examples.png` (Page 30)
**Chatbot launch button contextual examples**

Different launch implementations:
- Masthead items
- Icon buttons in corner
- Menu toggles
- "Chat with AI" buttons with dropdown
- Various positioning options

Shows flexibility while maintaining consistent robot icon.

### `chatbot-avatar-donts.png` (Page 31)
**Chatbot avatar don'ts - Part 1**

Three things NOT to do:
1. ❌ Don't use other icons/robots as chatbot avatars
2. ❌ Don't use robot icons for non-chatbot AI features
3. ❌ Don't use robot icons to represent humans

### `chatbot-donts-gradients.png` (Page 32)
**Chatbot avatar don'ts - Part 2**

Three more things NOT to do:
1. ❌ Don't use gradients on launch buttons
2. ❌ Don't use robot variations for state changes/alerts
3. ❌ Don't change robot color based on emotion

## Chat Message Styling

### `chat-messages-do-control-token.png`
**Do this: Use control tokens for chat message backgrounds**

✅ Correct example: Chat message boxes using standard PatternFly control tokens for background colors.

### `chat-message-dont-gradients.png`
**Don't use gradients on chat message boxes**

❌ Incorrect: Gradients cannot be used on the message box. Do not use gradients to indicate "thinking" or progress states. The chat conversation area has its own dedicated animation for those interactions.
