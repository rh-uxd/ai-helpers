# AI/ML engineer / developer

## Who

Builds and integrates serving, SDKs, and application code around models — often as a full-stack AI practitioner (selection, RAG, evals, integration). Expects predictable APIs, logs, errors, and developer documentation. Comfortable reading technical detail. Prefer `app-developer` when the audience is general application developers, not AI/ML builders.

## When to use

- Audience is AI engineers, ML engineers, or developers building AI applications
- Flows involving model playgrounds, APIs, SDKs, RAG/MCP, endpoints, or integration setup
