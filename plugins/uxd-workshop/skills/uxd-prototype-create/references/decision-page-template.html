Layout Pattern

What overall page structure best serves operators browsing and inspecting items?

  • All
  • 1. Layout
  • 2. Interaction
  • 3. Density

Option A

Card Grid

Summary cards in a responsive grid; click through to detail.

Pros
  • Scannable discovery
  • Works well on wide screens
Cons
  • Weak for dense ops tables
  • Detail requires navigation

When to use: Catalog / discovery workflows.

Recommended

Option B

List + Detail

Master list on the left; detail panel on the right.

Pros
  • Browse + inspect without losing context
  • Maps to common admin patterns
Cons
  • Needs careful responsive collapse
  • Less visual for marketing-style content

When to use: Browse-heavy ops and configuration lists.

Option C

Dashboard

Widgets and status panels as the primary surface.

Pros
  • Status at a glance
Cons
  • Weak for item-level CRUD

When to use: Monitoring and operational overview.

Option D

Wizard / Stepper

Guided multi-step flow as the primary surface.

Pros
  • Reduces cognitive load for complex create
Cons
  • Poor fit for ongoing browse/manage

When to use: Complex creation or setup flows.
